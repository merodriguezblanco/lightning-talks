RVM

Martin Rodriguez

Introduction

It often happens that we are working on several rails projects and they use different versions of Ruby.

So, it would be very helpful to have a tool that allows us to easily switch the ruby version between projects!

Here is when Ruby Version Management (or RVM) comes to the resque! RVM is a command-line tool allows us to easily install, manage, and work with multiple versions of ruby.

Installation

            
curl -L https://get.rvm.io | bash -s stable
            
          
            
curl -L https://get.rvm.io | bash -s stable --ruby --rails

Installing RVM from the command-line is very simple using the curl command. We can also pass some flags as well.

For example, the --ruby flag installs the latest ruby version, and the --rails flag installs the latest rails version as well.

Usage

            
$ rvm [options] command
            
          
            
$ rvm help
            
          
            
$ rvm help list

Once we have RVM installed, we should have the rvm command available on the terminal

The syntax for the RVM command is the following: rvm [options] command

For example, rvm help displays information on how to use rvm

If we would like help on a specific command, we can reference it after the "help" to get more detailed instructions

$ rvm install 2.0.0-p576

Once we decide what version we want to install, we can install it with the following command: rvm install 2.0.0-p576

How to switch rubies

            
$ rvm list

   ruby-2.1.2 [ x86_64 ]

Once we have installed a few versions of Ruby, we can list them on the terminal with the rvm list command

As we can see, RVM gives us a handy guide to tell us which are the current and the default Ruby versions. In this case, we have 6 versions of ruby installed, and the 2.0.0-p576 is the version we are using for this project.

How to switch rubies

            
$ rvm use 2.2.3
Using /Users/Martin/.rvm/gems/ruby-2.2.3

Let's suppose we want to switch to a different version, we can do so by typing: rvm use 2.2.3

How to use gemsets

            
$ rvm gemset list
            
          
            
$ rvm gemset list_all

So far we know how to install different ruby versions using rvm. But it is also possible to install different versions of a gem for the same Ruby version. RVM provides gemsets, which basically are separate gem directories, so the gems in one gemset do not conflict with the gems in another gemset..

To see the available gemsets for the current Ruby, we can type: rvm gemset list

We can also see all the gems installed for all the Ruby versions on our machine by typing: rvm gemset list_all

We can create project-specific configurations that specify what Ruby version and gemset to use by creating a file with a specific name at the root of our project directory.

This eliminates the need to manually keep track of the ruby version we have active.

We can create this file with the name .rvmrc or .ruby-version

How to automate our environment

            
# .ruby-version
ruby-2.2.3


# .ruby-gemset
gemset-name

Inside this file, we can put the ruby version we want to use, and next time we navigate to the project directory this version of ruby will be automatically selected.

Conclusion

As we can see, RVM is a flexible tool that can be used to manage projects and entire Ruby environments. We can use RVM to configure development conditions, server installations, and even to deploy our application.

Alternatives to RVM

  • rbenv
  • chruby
  • ry

RVM is not the only ruby version management tool that exists, we can also choose another one that suits best our needs such as: rbenv, chruby or ry
